# Vision

This project aims to become a premium markdown writing environment rather than a simple markdown input box.

## Product Goal

Create Grain, a markdown editor that feels as fluid as Typora, as customizable as Atom, and as visually deliberate as a modern design tool.

## Core Experience

- Fast startup and responsive editing for short notes and long-form writing
- Excellent markdown ergonomics with strong keyboard support
- A reading and writing experience centered on typography and layout
- Flexible workspace modes: focused writing, split preview, and preview-first reading
- Customization that changes the product in meaningful ways, not just accent colors

## Must-Win Areas

- Editor quality: shortcuts, selections, markdown helpers, code block support
- Preview quality: faithful rendering, beautiful defaults, readable long documents
- Customization: themes, fonts, spacing, layout density, and panel visibility
- Reliability: autosave, draft recovery, stable rendering, predictable behavior
- Polish: motion, empty states, onboarding, and responsive behavior

## Product Pillars

### Writing Flow

Reduce friction between thought and output. Common actions should be available by keyboard, toolbar, and command palette.

### Visual Craft

The application should feel designed, not assembled. Typography, spacing, color, and motion are part of the product.

### Personalization

Users should be able to shape the workspace around how they write, review, and publish.

### Extensibility

The initial app can be compact, but the architecture should leave room for plugins, custom commands, and desktop packaging later.

## MVP Outcome

Deliver Grain as a markdown editor that someone could genuinely choose for daily writing because it already feels fast, attractive, and thoughtfully built.
